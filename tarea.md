# Proyecto Final Integrador (Capstone)

**Curso:** Ingeniería de Datos  
**Duración:** 5 días hábiles  
**Modalidad:** Individual o equipos de hasta tres integrantes.

---

# 1. Presentación

Bienvenido al proyecto final del curso.

Durante las últimas semanas aprendió sobre bases de datos, SQL, modelado de datos, procesos ETL, Python, Pandas, APIs, calidad de datos, Data Warehouse, BigQuery, Snowflake y visualización.

En este proyecto deberá integrar todos esos conocimientos en una única solución empresarial.

La finalidad es simular un proyecto real de consultoría donde usted ha sido contratado como Ingeniero de Datos para resolver un problema de negocio.

---

# 2. Contexto de la empresa

La empresa ficticia **DataCommerce GT** comercializa productos tecnológicos mediante tiendas físicas, comercio electrónico, ventas corporativas y WhatsApp.

Con el crecimiento de la empresa la información comenzó a almacenarse en diferentes sistemas, por lo que la gerencia ya no puede obtener reportes confiables ni oportunos.

Su misión será diseñar e implementar una plataforma de datos que centralice la información y permita generar indicadores para apoyar la toma de decisiones.

---

# 3. Problema empresarial

Actualmente los datos provienen de distintas fuentes, presentan errores, duplicados, formatos diferentes y registros incompletos.

Cada departamento genera reportes distintos y los resultados no coinciden.

La dirección necesita una única fuente de información confiable.

---

# 4. Objetivo general

Construir una solución completa de ingeniería de datos que integre información de múltiples fuentes, la transforme, la almacene en un Data Warehouse y genere información útil para la toma de decisiones.

---

# 5. Objetivos específicos

- Integrar archivos CSV, Excel, JSON, SQLite y una API REST simulada.
- Diseñar un modelo relacional y un modelo dimensional.
- Implementar un proceso ETL/ELT reproducible.
- Aplicar reglas de calidad de datos.
- Cargar la información al Data Warehouse.
- Realizar consultas SQL analíticas.
- Construir indicadores con Python y Pandas.
- Presentar conclusiones de negocio mediante gráficos e informe ejecutivo.

---

# 6. Fuentes de datos

Se entregarán diferentes archivos que representan la operación diaria de la empresa:

- Ventas (CSV)
- Productos (Excel)
- Clientes (JSON)
- Inventario (SQLite)
- Campañas de marketing (API REST simulada)

No se garantiza que los datos estén limpios.

Es responsabilidad del estudiante identificar y corregir los problemas encontrados.

---

# 7. Reglas del proyecto

- No modificar manualmente los archivos fuente.
- Todas las transformaciones deben realizarse mediante código Python o SQL.
- El pipeline debe poder ejecutarse nuevamente sin intervención manual.
- Documentar todas las decisiones técnicas.
- Justificar cada regla de limpieza implementada.

---

# 8. Actividades esperadas

- Explorar las fuentes de datos.
- Analizar la calidad de la información.
- Diseñar la arquitectura de la solución.
- Construir el modelo entidad-relación.
- Construir el modelo estrella.
- Desarrollar el proceso ETL.
- Cargar el Data Warehouse.
- Generar consultas SQL.
- Construir KPIs.
- Crear visualizaciones.
- Elaborar documentación técnica y ejecutiva.

---

# 9. Cronograma de trabajo

## Día 1

Comprender el negocio, revisar archivos, identificar problemas, diseñar arquitectura, modelo relacional y plan del ETL.

## Día 2

Extraer datos desde todas las fuentes, documentar hallazgos y desarrollar la limpieza y estandarización.

## Día 3

Integrar datos, construir el modelo dimensional, generar tablas de hechos y dimensiones y cargar el Data Warehouse.

## Día 4

Desarrollar consultas SQL, calcular KPIs con Pandas, validar resultados y construir gráficos.

## Día 5

Preparar documentación, dashboard, presentación ejecutiva y defensa técnica.

---

# 10. Entregables

- Arquitectura de la solución.
- Modelo entidad-relación.
- Modelo dimensional.
- Notebooks de extracción, transformación, carga y análisis.
- Scripts SQL.
- Diccionario de datos.
- Dashboard con KPIs.
- Informe ejecutivo.
- Presentación final.

---

# 11. Criterios de evaluación

- Integración de datos (15%)
- Calidad del ETL (20%)
- Modelado de datos (15%)
- SQL (10%)
- Análisis con Python (10%)
- KPIs y visualizaciones (10%)
- Documentación (10%)
- Presentación final (10%)

---

# 12. Recomendaciones

Trabaje de forma ordenada.

Mantenga una estructura clara de carpetas, documente cada paso y valide continuamente sus resultados.

Recuerde que en un entorno empresarial la claridad y la reproducibilidad son tan importantes como el funcionamiento del código.

---

# 13. Resultado esperado

Al finalizar la semana deberá existir una plataforma de datos completamente funcional capaz de integrar información de distintas fuentes, alimentar un Data Warehouse y producir indicadores confiables para la gerencia.

El proyecto debe demostrar que el estudiante puede desenvolverse como Ingeniero de Datos Junior en un entorno similar al de una empresa real.
